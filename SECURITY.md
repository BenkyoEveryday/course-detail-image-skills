# 安全政策

## 支持范围

安全修复只针对最新发布版本。请先确认问题在 `main` 或最新版本中仍然存在。

## 报告漏洞

请不要为安全问题创建公开 Issue。优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私下报告；如果该入口不可用，请联系仓库维护者。

报告中请包含：

- 受影响的版本与文件；
- 可复现的最小步骤；
- 可能泄露或被修改的数据；
- 建议的缓解方式（如有）。

## 敏感数据

本仓库不应包含客户 Word、课件截图、学生信息、生成结果、访问令牌、Cookie、账号信息或私有素材。发现误提交时，请立即停止传播并联系维护者处理；仅在后续提交中删除文件并不能从 Git 历史中移除敏感内容。
